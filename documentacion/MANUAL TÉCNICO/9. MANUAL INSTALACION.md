## Introducción

Este manual describe el proceso de instalación y despliegue de **FAMA** en un servidor Ubuntu en producción. La aplicación se despliega como contenedores Docker orquestados con Docker Compose, siendo nginx el proxy inverso frontal.

**Arquitectura**:
- **Servidor frontal**: nginx (puertos 80/443)
- **Aplicación**: Flask + Gunicorn en contenedor Docker (puerto 5000 interno)
- **Base de datos**: MongoDB en contenedor Docker (puerto 27017 interno, no expuesto)
- **Almacenamiento**: Volumen persistente para datos MongoDB

## Requisitos Previos

### Hardware

- **CPU**: Mínimo 2 cores, recomendado 4+
- **RAM**: Mínimo 2 GB, recomendado 4+ GB
- **Almacenamiento**: Mínimo 20 GB, recomendado 50+ GB (depende del volumen de datos esperado)
- **Conexión de red**: Ancho de banda suficiente para el tráfico previsto

### Software

- **Sistema Operativo**: Ubuntu Server 20.04 LTS o superior
- **Docker Engine**: 20.10 o superior
- **Docker Compose**: 2.0 o superior
- **Nginx**: 1.18 o superior (instalado en el host, no en Docker)
- **Git**: Para clonar el repositorio (opcional si se sube el código manualmente)

### Acceso

- Acceso SSH como usuario con permisos de sudo
- Acceso a internet para descargar imágenes de Docker
- Un dominio apuntando a la IP del servidor (recomendado)

## Arquitectura de Despliegue

```
Internet (80/443)
        ↓
   Nginx (Host)
   Proxy Inverso
   Terminación TLS
        ↓
Docker Network
        ↓
    ┌───┴──────┐
    ↓          ↓
 Web Service  MongoDB
 Gunicorn     Contenedor
 Puerto 5000  Puerto 27017
              (no expuesto)
```

### Flujo de Peticiones

1. El cliente accede a través del dominio (puerto 80/443)
2. Nginx recibe la petición y realiza la terminación TLS
3. Nginx reenvía internamente a la aplicación Flask (127.0.0.1:5000)
4. Flask consulta MongoDB a través de la red interna de Docker
5. La respuesta vuelve por el mismo camino al cliente

## Preparación del Servidor

### 1. Actualizar el Sistema

```bash
sudo apt update
sudo apt upgrade -y
```

### 2. Instalar Docker Engine

#### Opción A: Repositorio oficial de Docker

```bash
# Instalar dependencias
sudo apt install -y ca-certificates curl gnupg lsb-release

# Añadir la clave GPG de Docker
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Añadir el repositorio de Docker
echo \
  "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Actualizar e instalar Docker Engine
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

#### Opción B: Repositorio de Distribución

```bash
sudo apt install -y docker.io docker-compose
```

### 3. Configurar Docker para Usuario No-Root

```bash
# Crear grupo docker (si no existe)
sudo groupadd docker || true

# Añadir el usuario actual al grupo docker
sudo usermod -aG docker $USER

# Activar cambios de grupo
newgrp docker

# Verificar instalación
docker run hello-world
```

### 4. Habilitar Docker en el Arranque

```bash
sudo systemctl enable docker
sudo systemctl start docker
```

### 5. Instalar Nginx

```bash
sudo apt install -y nginx curl
```

```bash
# Crear directorio para la aplicación
sudo mkdir -p /home/fama/FAMA
sudo chown -R $USER:$USER /home/fama/FAMA

# Crear subdirectorios para certificados
```

### 7. Verificar Puertos Disponibles


## Despliegue de FAMA

### 1. Clonar o Descargar el Repositorio

```bash
cd /home/fama/FAMA

# Opción A: Clonar desde GitHub
git clone https://github.com/hondara91/FAMA.git

# Opción B: O descargar y descomprimir manualmente
# Copiar archivos al directorio /home/fama/FAMA
```

### 2. Configurar Variables de Entorno (opcional, al trabajar con Docker Compose no es necesario)

Crear archivo `.env` en el directorio raíz de FAMA:

```bash
cat > /home/fama/fama.env << 'EOF'
# Seguridad
SECRET_KEY=CAMBIAR_POR_UNA_CLAVE_FUERTE_Y_ALEATORIA_DE_MINIMO_32_CARACTERES

# Base de datos
MONGO_URI=mongodb://mongo:27017/
MONGO_DB_NAME=fama_db

# Flask
FLASK_ENV=production
FLASK_DEBUG=0
FLASK_HOST=0.0.0.0
FLASK_PORT=5000

# Limits
MAX_CONTENT_LENGTH=5242880
EOF
```

**Importante**: Generar una clave `SECRET_KEY` fuerte:

```bash
# Opción 1: Usar openssl
openssl rand -hex 32

# Opción 2: Usar Python
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### 3. Preparar Docker Compose para Producción

Crear `docker-compose.prod.yml` en `/opt/fama`:

```bash
cat > /home/fama/FAMA/docker-compose.prod.yml << 'EOF'
version: '3.8'

services:
  web:
    build: .
    container_name: fama_web
    restart: always
    environment:
      FLASK_ENV: ${FLASK_ENV}
      FLASK_DEBUG: ${FLASK_DEBUG}
      FLASK_HOST: ${FLASK_HOST}
      FLASK_PORT: ${FLASK_PORT}
      SECRET_KEY: ${SECRET_KEY}
      MONGO_URI: ${MONGO_URI}
      MONGO_DB_NAME: ${MONGO_DB_NAME}
    depends_on:
      mongo:
        condition: service_healthy
    networks:
      - fama_network
    # NO exponemos puertos al host (solo accesible vía Docker network)
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

  mongo:
    image: mongo:4.4
    container_name: fama_mongo
    restart: always
    environment:
      MONGO_INITDB_DATABASE: ${MONGO_DB_NAME}
    volumes:
      - mongo_data:/data/db
    networks:
      - fama_network
    # NO exponemos puertos (solo accesible vía Docker network)
    healthcheck:
      test: ["CMD", "mongo", "--quiet", "--eval", "db.adminCommand('ping').ok"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

volumes:
  mongo_data:
    driver: local

networks:
  fama_network:
    driver: bridge
EOF
```

### 4. Construir e Iniciar Contenedores

```bash
cd /home/fama/FAMA

# Construir la imagen
docker compose -f docker-compose.prod.yml build

# Iniciar los servicios
docker compose -f docker-compose.prod.yml up -d

# Verificar estado
docker compose -f docker-compose.prod.yml ps
docker compose -f docker-compose.prod.yml logs web
```

### 5. Crear Usuario Admin Inicial

```bash
# Ejecutar script de creación de admin en el contenedor
docker exec fama_web python scripts/crear_admin.py
```

### 6. Importar Datos de Prueba (Opcional)

```bash
# Ejecutar script de importación de datos
docker exec fama_web python scripts/importar_seed_data.py
```

## Configuración de Nginx

### 1. Crear Configuración de Nginx

Crear `/etc/nginx/sites-available/fama`:

```bash
sudo cat > /etc/nginx/sites-available/fama << 'EOF'
upstream fama_backend {
    server 127.0.0.1:5000;
}

# Redirigir HTTP a HTTPS
server {
    listen 80;
    listen [::]:80;
    server_name 192.168.7.80;
    
    # Permitir renovación de certificados Let's Encrypt
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }
    
    # Redirigir todo a HTTPS
    location / {
        return 301 https://$host$request_uri;
    }
}

# Configuración HTTPS (se actualiza con certificados reales)
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name 192.168.7.80;

    # Certificados SSL (ver sección siguiente)
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # Configuración SSL recomendada
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # Headers de seguridad
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Logs
    access_log /home/fama/FAMA/logs/nginx_access.log;
    error_log /home/fama/FAMA/logs/nginx_error.log;

    # Tamaño máximo de cuerpo de petición
    client_max_body_size 5M;

    location / {
        proxy_pass http://fama_backend;
        proxy_http_version 1.1;
        
        # Headers de proxy
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $server_name;
        
        # WebSocket support
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Servir archivos estáticos directamente (opcional, optimización)
    location /static/ {
        alias /home/fama/FAMA/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # Bloquear acceso a directorios sensitivos
    location ~ /\. {
        deny all;
    }
}
EOF
```

### 2. Habilitar el Sitio

```bash
# Crear enlace simbólico
sudo ln -s /etc/nginx/sites-available/fama /etc/nginx/sites-enabled/

# Eliminar configuración default si está habilitada
sudo rm -f /etc/nginx/sites-enabled/default

# Verificar sintaxis
sudo nginx -t

# Recargar Nginx
sudo systemctl reload nginx
```

### 3. Verificar Conectividad

```bash
# Desde el host, verificar que puede conectar a la aplicación
curl http://localhost:5000/

# O si Nginx está configurado
curl http://localhost/
```

## Configuración de SSL/TLS

### Opción A: Certificados Let's Encrypt con Certbot (Recomendado)

```bash
# Instalar Certbot
sudo apt install -y certbot python3-certbot-nginx

# Solicitar certificado
sudo certbot certonly --webroot \
  -w /var/www/certbot \
  -d 192.168.7.80 \
  -d 192.168.7.80 \
  --non-interactive --agree-tos -m admin@192.168.7.80

# Actualizar configuración nginx con rutas reales
sudo sed -i 's|/etc/nginx/ssl/cert.pem|/etc/letsencrypt/live/mi-dominio.com/fullchain.pem|g' /etc/nginx/sites-available/fama
sudo sed -i 's|/etc/nginx/ssl/key.pem|/etc/letsencrypt/live/mi-dominio.com/privkey.pem|g' /etc/nginx/sites-available/fama

# Verificar y recargar
sudo nginx -t
sudo systemctl reload nginx

# Configurar renovación automática
sudo systemctl enable certbot.timer
sudo systemctl start certbot.timer
```

### Opción B: Certificados Autofirmados (Desarrollo/Testing)

```bash
# Crear certificado autofirmado
sudo openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/key.pem \
  -out /etc/nginx/ssl/cert.pem \
  -subj "/C=ES/ST=State/L=City/O=Organization/CN=localhost"

# Establecer permisos
sudo chmod 600 /etc/nginx/ssl/key.pem
sudo chmod 644 /etc/nginx/ssl/cert.pem

# Recargar Nginx
sudo systemctl reload nginx
```

## Copias de Seguridad

### 1. Backup Manual de MongoDB

```bash
# Crear directorio de backups
mkdir -p /home/fama/FAMA/backups/$(date +%Y%m%d)

# Hacer backup del volumen MongoDB
docker exec fama_mongo mongodump \
  --out /data/db/backup_$(date +%Y%m%d_%H%M%S)

# Copiar a ubicación persistente
docker cp fama_mongo:/data/db/backup_* /opt/fama/backups/
```

### 2. Script de Backup Automatizado

Crear `/opt/fama/backup.sh`:

```bash
#!/bin/bash

BACKUP_DIR="/opt/fama/backups"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_PATH="$BACKUP_DIR/fama_backup_$DATE"

mkdir -p "$BACKUP_PATH"

# Backup de MongoDB
docker exec fama_mongo mongodump --out /tmp/backup_$DATE
docker cp fama_mongo:/tmp/backup_$DATE "$BACKUP_PATH/mongodb"

# Backup de ficheros subidos
cp -r /home/fama/FAMA/static/uploads "$BACKUP_PATH/uploads" || true

# Compresión
tar -czf "$BACKUP_PATH.tar.gz" -C "$BACKUP_DIR" "fama_backup_$DATE"
rm -rf "$BACKUP_PATH"

# Mantener solo últimos 7 backups
find "$BACKUP_DIR" -name "fama_backup_*.tar.gz" -type f -mtime +7 -delete

echo "Backup completado: $BACKUP_PATH.tar.gz"
```

### 3. Programar Backups con Cron

```bash
# Editar crontab
crontab -e

# Añadir línea para ejecutar backup diariamente a las 2 AM
0 2 * * * /home/fama/FAMA/backup.sh >> /opt/fama/logs/backup.log 2>&1
```

### 4. Restaurar desde Backup

```bash
# Descomprimir
tar -xzf fama_backup_20240610_120000.tar.gz -C /opt/fama/backups/

# Detener servicios
docker compose -f docker-compose.prod.yml down

# Restaurar MongoDB
docker run --rm -v fama_mongo_data:/data/db \
  -v /home/fama/FAMA/backups/fama_backup_20240610_120000/mongodb:/backup \
  mongo:4.4 mongorestore /backup

# Reiniciar servicios
docker compose -f docker-compose.prod.yml up -d
```

## Troubleshooting

### Problema: La aplicación no está accesible

```bash
# Verificar estado de contenedores
docker compose -f docker-compose.prod.yml ps

# Ver logs de errores
docker compose -f docker-compose.prod.yml logs web

# Verificar conectividad interna
docker exec fama_web curl http://localhost:5000/

# Verificar que Nginx está corriendo
sudo systemctl status nginx
```

### Problema: Error de conexión a MongoDB

```bash
# Verificar que MongoDB está corriendo
docker compose -f docker-compose.prod.yml ps mongo

# Ver logs de MongoDB
docker compose -f docker-compose.prod.yml logs mongo

# Verificar conectividad desde aplicación
docker exec fama_web python -c "from utils.db import check_mongo_connection; print(check_mongo_connection())"
```

### Problema: Certificado SSL no válido

```bash
# Verificar estado de certbot
sudo certbot certificates

# Forzar renovación
sudo certbot renew --force-renewal

# Verificar permisos de archivo
sudo ls -la /etc/letsencrypt/live/
```

### Problema: Puerto 80/443 ya en uso

```bash
# Identificar proceso usando el puerto
sudo lsof -i :80
sudo lsof -i :443

# Detener el servicio conflictivo
sudo systemctl stop <nombre_servicio>

# O cambiar puerto de Nginx (no recomendado)
```

### Problema: Espacio en disco insuficiente

```bash
# Ver uso de disco
df -h

# Limpiar volúmenes de Docker
docker volume prune

# Limpiar logs antiguos
sudo journalctl --vacuum=100M

# Ver tamaño de carpetas
du -sh /home/fama/FAMA/*
du -sh /var/lib/docker/volumes/
```

## Seguridad en Producción

### 1. Firewall

```bash
# Habilitar firewall
sudo ufw enable

# Permitir SSH
sudo ufw allow ssh

# Permitir HTTP y HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Verificar reglas
sudo ufw status
```

### 2. Usuarios y Permisos

```bash
# Crear usuario específico para FAMA
sudo useradd -r -s /bin/false fama_user

# Cambiar propietario de archivos
sudo chown -R fama_user:fama_user /opt/fama

# Establecer permisos correctos
sudo chmod -R 750 /home/fama/FAMA
sudo chmod 600 /home/fama/FAMA/.env
```

### 3. Límites de Recursos

En `docker-compose.prod.yml`, añadir limitaciones:

```yaml
services:
  web:
    # ... resto de configuración
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
  
  mongo:
    # ... resto de configuración
    deploy:
      resources:
        limits:
          cpus: '1'
          memory: 2G
```

### 4. Actualizaciones de Seguridad

```bash
# Actualizar el sistema regularmente
sudo apt update && sudo apt upgrade -y

# Actualizar imágenes de Docker
docker pull mongo:4.4
docker pull python:3.14-slim

# Reconstruir contenedores con nuevas imágenes
docker compose -f docker-compose.prod.yml build --no-cache
docker compose -f docker-compose.prod.yml up -d
```

### 5. Secretos Seguros

- Nunca incluir `SECRET_KEY` en el repositorio
- Usar variables de entorno en el servidor
- Usar herramientas como HashiCorp Vault para gestionar secretos
- Rotación regular de claves

