# 5. ALGORITMOS RELEVANTES

Este documento explica, en un lenguaje sencillo, los algoritmos y procesos más relevantes del proyecto FAMA. No se trata de fórmulas complejas, sino de las ideas que guían cómo funciona la aplicación internamente.

## 5.1 Autenticación de usuarios y control de acceso

### ¿Qué hace?
El sistema valida que cada usuario que intenta entrar sea quien dice ser y le da permiso solo a las páginas que le corresponden.

### Cómo funciona
- El usuario envía sus datos y contraseña en el formulario de login.
- La aplicación compara la contraseña ingresada con la contraseña almacenada en la base de datos.
- Para mayor seguridad, la contraseña se guarda como un valor "cifrado" (hash), de forma que nadie pueda verla en texto claro.
- Si las credenciales son correctas, la aplicación crea una sesión que mantiene al usuario conectado mientras navega.
- Si no son correctas, se bloquea el acceso y se muestra un mensaje de error.

### Por qué es importante
Este algoritmo protege las cuentas y evita que personas no autorizadas accedan a datos privados o funciones administrativas.

## 5.2 Validación de formularios

### ¿Qué hace?
Verifica que la información que el usuario ingresa en formularios sea correcta y completa antes de guardarla.

### Cómo funciona
- Cada formulario del proyecto tiene reglas básicas: campos obligatorios, formatos válidos, rangos numéricos, etc.
- El sistema revisa los datos al momento de enviarlos.
- Si falta algún campo o hay un valor inválido, el formulario vuelve a la página con un mensaje que explica el error.
- Sólo si todo está bien, el sistema guarda la información en la base de datos.

### Ejemplos típicos
- Comprobar que el correo tenga el formato correcto.
- Verificar que el precio de un anuncio de vivienda sea un número válido.
- Confirmar que una contraseña nueva tenga la longitud mínima requerida.

## 5.3 Filtrado y búsqueda de contenido

### ¿Qué hace?
Permite encontrar anuncios, servicios, eventos y temas de foro de forma rápida, usando criterios como categoría, ubicación o texto.

### Cómo funciona
- El usuario puede elegir opciones de búsqueda o filtros en la interfaz.
- El sistema traduce esas opciones en una consulta a la base de datos.
- La base de datos devuelve solo los resultados que coinciden con los criterios.
- Los resultados se muestran ordenados y, cuando hay muchos, se presentan en páginas para no sobrecargar la vista.

### Por qué es importante
Mejora la experiencia del usuario al encontrar sólo lo que le interesa, sin navegar entre listados muy largos.

## 5.4 Manejo de archivos subidos

### ¿Qué hace?
Controla cómo se reciben, guardan y muestran los archivos que los usuarios suben, como fotos de anuncios o documentos.

### Cómo funciona
- El usuario selecciona un archivo y lo envía desde el formulario.
- El sistema verifica que el archivo sea un tipo permitido (por ejemplo imagen o PDF) y no supere el tamaño máximo.
- Si el archivo es válido, se guarda en una carpeta específica del servidor.
- Se genera un nombre seguro para evitar conflictos o problemas de seguridad.
- El sistema registra la ruta del archivo en la base de datos para poder acceder a él luego.

### Por qué es importante
Evita subir archivos peligrosos o corruptos, y mantiene organizados los recursos asociados a cada anuncio o perfil.

## 5.5 Gestión de roles y permisos

### ¿Qué hace?
Determina qué acciones puede realizar cada usuario según su perfil: administrador o usuario estándar.

### Cómo funciona
- El sistema asigna un rol a cada cuenta (por ejemplo, administrador o usuario normal).
- Antes de ejecutar funciones sensibles, se comprueba el rol del usuario actual.
- Si el usuario no tiene permiso, se le impide continuar y se le redirige o se le muestra un aviso.

### Ejemplos típicos
- Solo el administrador puede ver y gestionar logs o reportes globales.
- Un usuario estándar puede crear y editar sus propios anuncios, pero no los de otros.

## 5.6 Registro de actividad y trazabilidad

### ¿Qué hace?
Lleva un historial de acciones importantes realizadas en el sistema para poder revisarlas después.

### Cómo funciona
- Cada vez que ocurre un evento relevante (por ejemplo, un usuario edita un anuncio o se crea un reporte), se guarda una entrada en los registros.
- Cada entrada incluye datos como quién hizo la acción, cuándo y qué ocurrió.
- Los administradores pueden consultar estos registros para revisar el comportamiento del sistema.

### Por qué es importante
Ayuda a detectar problemas, auditorías y a comprender cómo se ha usado la aplicación.

## 5.7 Flujo general de las operaciones principales

En el proyecto, la mayoría de las acciones sigue un mismo patrón sencillo:

1. El usuario interactúa con la interfaz (clic, enviar formulario, buscar un elemento).
2. El servidor recibe la solicitud y aplica validaciones.
3. El servidor consulta o actualiza la base de datos según corresponda.
4. El servidor devuelve una respuesta o redirige al usuario.

Este flujo se aplica a tareas como:
- iniciar sesión,
- publicar o editar un anuncio,
- responder un post del foro,
- reservar o solicitar un servicio.

## 5.8 Conclusión

Aunque no todos los algoritmos son matemáticos o complejos, los procesos descritos son los más relevantes para que el proyecto funcione de manera segura, ordenada y útil para los usuarios.

Comprenderlos ayuda a saber por qué el sistema pide ciertos datos, por qué hay restricciones y cómo se protege la información dentro de FAMA.
